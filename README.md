# 库存优化思路（报童模型）

## 1. 问题目标
为每个 `(item_id, store_code)` 预测 **2015‑12‑14 ~ 2015‑12‑27**（两周）的最优库存量，使得总成本最小。

- **超储成本** `a`（补多）
- **欠储成本** `b`（补少）

## 2. 模型选择
**报童模型**：最优库存量 \( Q^* \) 满足  
\[
F(Q^*) = \frac{b}{a+b}
\]  
假设需求正态分布 → \( Q^* = \mu + z \cdot \sigma \)，其中 \( z = \Phi^{-1}\left(\frac{b}{a+b}\right) \)

**核心任务**：估计未来两周需求的均值 \( \mu \) 和标准差 \( \sigma \)。

## 3. 数据处理
- 删除所有年份的 **11月11日** 和 **12月12日** 数据（促销异常点）。
- 按 ISO 周（周一始）聚合为每周需求。
- 对连续两周求和，构建 **两周需求滚动块**（块结束周为第二周）。

## 4. 预测 \( \mu \) 和 \( \sigma \)
- **近期块**：距离目标结束周 ≤12 周，指数加权（半衰期约 2.8 周）。
- **季节块**：相同 ISO 周编号、更早年份的块。
- 融合规则：
  - 仅有近期 → 取加权平均
  - 仅有季节 → 取季节均值
  - 两者都有 → 70% 近期 + 30% 季节
- 标准差来自近期/历史块的样本标准差，稀疏时用 \( \sqrt{\mu} \times 1.5 \)。

## 5. 后备机制（分层）
当某 `(item,store)` 数据不足：
- 使用**商品级**周需求均值和标准差（跨店铺）
- 若商品级也无数据 → 全局均值和标准差
- 两周预测：\( \mu = 2 \times \mu_{1w},\ \sigma = \sqrt{2} \times \sigma_{1w} \)（独立周假设）

## 6. 最终输出
- 对每个配置中的 `(item_id, store_code)` 计算 \( Q^* \)。
- 保存为 `target_inventory.csv`（列：item_id, store_code, target）。
